# Wiki Overlap Map

The overlap map explains which wiki owns each StegTalk-adjacent concept and how cross-wiki links should be interpreted.

## StegTalk Wiki Owns

- product/runtime behavior
- message lifecycle
- contact routing
- local inbox projection
- local persistence
- public discovery runtime
- shell state and shell actions
- local account profile and account session behavior

## StegGuardian Wiki Owns

- guardian boundaries
- recovery authority
- account federation
- device-bound guardian enforcement
- account protection concepts
- guardian/account vocabulary

Primary linked repo: `StegVerse-002/stegguardian-wiki`

## Admissibility Wiki Owns

- standing
- authority
- execution validity
- transition admissibility
- guardian action admissibility
- recovery transition admissibility
- account federation admissibility

Primary linked repo: `StegVerse-Labs/admissibility-wiki`

## Site Owns

- public explanations
- concept summaries
- demos
- navigation
- publication records

Primary linked repo: `StegVerse-Labs/Site`

## Boundary

Overlap does not merge authority. Each wiki owns a different interpretive layer, and linked pages must preserve their own boundary status.
